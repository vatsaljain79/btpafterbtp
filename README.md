# 🎵 Shazam — Audio Fingerprinting Engine

A from-scratch Python implementation of the **Wang (2003) audio fingerprinting algorithm** — the same technique that powers Shazam. Identifies a short, noisy audio clip against a database of songs in **~20 ms**, achieving **100% accuracy** on 20 songs with 6 dB SNR noise.

> **Reference:** Avery Li-Chun Wang, *"An Industrial-Strength Audio Search Algorithm"*, ISMIR 2003.

---

## 🚀 Quick Start

```bash
# 1. Install dependencies
pip install numpy scipy matplotlib

# 2. Generate synthetic songs (20 songs, 30 s each)
python generate_songs.py

# 3. Build the fingerprint database
python database.py

# 4. Generate noisy query clips
python generate_query.py

# 5. Identify a query clip
python shazam.py identify query/demo_query.wav

# 6. Benchmark all 20 query clips
python shazam.py benchmark

# 7. Generate visualizations (optional)
python visualize.py
```

## 🧠 Algorithm Deep Dive

The algorithm runs in three stages: **fingerprinting**, **indexing**, and **matching**.

### Stage 1 — Spectrogram

Each audio file is converted to a Short-Time Fourier Transform (STFT) magnitude spectrogram.

| Parameter       | Value            |
|----------------|-----------------|
| Sample rate    | 8,000 Hz (mono) |
| FFT window     | 4,096 samples   |
| Hop size       | 512 samples     |
| Frequency bins | 2,049           |

```
Audio samples  →  STFT  →  |magnitude|  →  dB scale  →  spectrogram
```

### Stage 2 — Constellation Map

Local maxima (peaks) are extracted from the spectrogram. A point `(t, f)` is a peak if it has higher energy than all neighbors in a 20×20 bin neighborhood **and** exceeds a minimum dB threshold.

The sparse set of `(time_frame, freq_bin)` coordinates forms a **"constellation map"** — like stars in a night sky. These peaks are robust: they survive noise, EQ changes, codec compression, and bandwidth reduction because the highest-energy spectral features are least affected by distortion.

```
Spectrogram  →  local_max filter  →  amplitude threshold  →  density cap
→  [(t0,f0), (t1,f1), ...]   ← constellation map
```

### Stage 3 — Combinatorial Hashing

Each peak can serve as an **anchor point**. For every anchor, points within a **target zone** (Δt ∈ [1, 20] frames ahead, Δf ∈ ±100 bins) are paired with it. Each pair produces a 32-bit hash:

```
hash = (f_anchor << 22) | (f_target << 12) | Δt
```

With a fan-out `F = 10`, each anchor generates up to 10 hashes. This boosts entropy from ~10 bits (single peak) to ~30 bits per hash — making accidental hash collisions roughly **1,000,000× less likely**, dramatically accelerating the search.

```
Constellation map  →  anchor × target-zone pairs  →  32-bit hashes
Each hash → (hash32, anchor_time_frame)
```

**Storage:** Each record is a 64-bit struct: 32 bits for the hash + 32 bits for `(time_offset, track_id)`.

**Speed tradeoff (from the paper):**
- 10× more storage (fan-out F = 10)
- ~10,000× faster search than single-peak matching
- Negligible loss in detection probability

### Stage 4 — Searching & Scoring

To identify a query:

1. Fingerprint the query → list of `(hash, t_sample)` pairs
2. For every hash, look it up in the database → `[(track_id, t_db), ...]`
3. For each matching pair, compute `Δt = t_db − t_sample`
4. Group by `track_id` and histogram the `Δt` values
5. A **sharp peak** in the histogram means the track's features align perfectly with the query at a specific time offset — this is the match
6. The score = the height of that peak (number of coherently aligned hash pairs)

```
Query hashes  →  database lookup  →  Δt pairs per track
→  histogram per track  →  peak detection  →  ranked matches
```

**Why this works:** Spurious hash collisions produce Δt values scattered uniformly. But matching hashes all share the same Δt = `t_db_start − t_sample_start`, so they pile up at exactly one histogram bin.

## 🔧 Configuration

All key parameters are defined at the top of `fingerprint.py`:

```python
FFT_WINDOW_SIZE  = 4096   # samples per FFT frame
FFT_HOP_SIZE     = 512    # overlap hop
PEAK_NEIGHBORHOOD = 20    # local-max filter radius (bins)
MIN_AMPLITUDE_DB  = -60   # floor below which peaks are ignored
MAX_PEAKS_PER_SEC = 15    # density cap (peaks/second)
FAN_OUT           = 10    # target-zone pairings per anchor
TARGET_T_MIN      = 1     # target zone: min Δt (frames)
TARGET_T_MAX      = 20    # target zone: max Δt (frames)
TARGET_F_MIN      = -100  # target zone: min Δf (bins)
TARGET_F_MAX      = 100   # target zone: max Δf (bins)
```

**Tuning tips:**
- Increase `FAN_OUT` for better noise robustness at the cost of storage and speed.
- Increase `PEAK_NEIGHBORHOOD` to reduce false peaks in dense spectrograms.
- Lower `MAX_PEAKS_PER_SEC` for faster (but potentially less robust) indexing.
- For radio-quality audio, reduce `FAN_OUT` to 3–5 for sub-millisecond queries.
